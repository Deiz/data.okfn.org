Open Reference Data Key datasets in high quality, easy-to-use and open form

High Quality Essential Data

We have sourced, normalized and quality checked a set of key reference datasets such as country codes, currencies, GDP and population.

Standard Form, Bulk Access

All the datasets are provided in a standardized form and can be accessed in bulk as CSV together with a simple JSON schema.

Versioned & Packaged

All data comes from high quality source data packages and is versioned using git so that all changes are visible and datasets can be collaboratively maintained.

Datasets

Data Packages

All datasets start in source form as "data packages" with these stored in a git repo on GitHub. Data packages are designed to be really simple but to have just enough structure to be usable by tools. You can check out an existing, exemplar data package (GDP) on GitHub here.

Structure

A Data Package has the following structure on disk:

  # README in markdown format. Used to generate a short summary (first sentence),
  # longer summary (first paragraph) and general description (complete file)
  README.md
  # metadata and data schemas for this data package
  datapackage.json
  # directory for all data files (CSV)
  data/
  ..../my-data-file.csv
  ..../...more...files...
  # directory for scripts for processing and cleaning data
  scripts/
  ......./process.py

datapackage.json

datapackage.json is the central file in a Data Package as it provides both general metadata and schema information in a structured form that is machine usable. It is a JSON file and full details of its structure are in the data packages specification.

Data files

All data must be provided as CSV files (UTF8 encoded) and there should be only one data file in a data package.
